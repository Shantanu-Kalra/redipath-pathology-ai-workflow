# Redipath Pathology AI Workflow

An automated oncology report generation system built during my internship at **Redipath**, simulating an AI-assisted pathology pipeline for prostate core biopsy specimens.

---

## The Problem

In clinical pathology labs, report preparation is a time-intensive, manual process. Pathologists must cross-reference patient EHR records, interpret slide findings, and draft structured diagnostic reports — often under high caseload. This project simulates how an AI agent layer can automate the routine parts of that workflow, allowing pathologists to focus on final clinical judgment.

---

## What This System Does

The workflow simulates the end-to-end journey of a biopsy specimen through a clinical AI pipeline:

```
Specimen Received → EHR Data Retrieval → AI Clinical Summary
       → AI Slide Analysis → Draft Report Generation
              → Pathologist Review Queue
```

1. **EHR Integration** — A mock Electronic Health Record system retrieves patient history (PSA levels, family history, comorbidities)
2. **NLP Extraction** — The AI agent parses unstructured EHR text using regex-based NLP (`stringr`) to extract clinically relevant values
3. **Slide Analysis Simulation** — The agent produces structured findings (Gleason score, tumor quantification, perineural invasion status) based on specimen type
4. **Report Drafting** — A structured pathology report is generated using `glue` templating, formatted for pathologist sign-off
5. **LIS Audit Trail** — A mock Laboratory Information System logs every status change for traceability

> **Note on data privacy:** All patient data in this project is fully simulated (dummy records only). Real patient data was never used, in compliance with HIPAA and clinical data governance standards.

---

## Architecture

The project is split into four modular R files, each representing a distinct system component:

| File | Role |
|------|------|
| `data_models.R` | `Specimen` R6 class — lifecycle state management for biopsy specimens |
| `simulated_systems.R` | `MockEHRSystem` and `MockLIS` R6 classes — simulates clinical backend systems |
| `ai_agent.R` | `PathologyAIAgent` R6 class — NLP extraction, slide analysis, and report drafting |
| `main.R` | Orchestration — wires all components together and runs the full workflow |

This separation of concerns mirrors how real clinical AI systems are architected: data models, system integrations, and AI logic as independent layers.

---

## Tech Stack

- **Language:** R
- **OOP Framework:** R6 (object-oriented class system)
- **NLP / Text Processing:** `stringr`
- **Report Templating:** `glue`

---

## Responsible AI Design

A deliberate product decision is embedded in the system: every generated report carries an explicit disclaimer —

> *"This is an AI-generated draft. All findings and diagnoses require final review, verification, and sign-out by a qualified pathologist."*

This reflects a core principle of deploying AI in regulated clinical environments: **AI augments, but does not replace, clinical judgment.** The system is designed to accelerate the workflow, not automate the final decision.

## How to Run

**Prerequisites:** R (≥ 4.0), with the following packages installed:

```r
install.packages(c("R6", "stringr", "glue"))
```

**Run the workflow:**

```r
source("main.R")
run_workflow()
```

This will print the full simulated workflow — EHR retrieval, AI agent logs, slide findings, and the final AI-drafted pathology report — to the console.
